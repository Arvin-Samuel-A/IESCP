# Wireframes – IESCP

This document provides an overview of the **low‑fidelity wireframes** created for the project **IESCP (Influencer–Sponsor Engagement & Campaign Platform)**. The wireframes were designed to demonstrate core user flows, screen layout, and role‑based functionality, without focusing on visual styling or branding.

## Screen Overview

A total of **6 low‑fidelity desktop screens** were created, following a consistent layout system (1440px width, left navigation, top header where applicable).

---

### 1. Login Screen

**Purpose:**
- Acts as the entry point for all users (Admin, Sponsor, Influencer).

**Description:**
- Centered layout with application title and subtitle
- Username and password input fields
- Login action button
- Link to create a new account

This screen establishes authentication as the first step before accessing any role‑specific functionality.

---

### 2. Sponsor Dashboard

**Purpose:**
- Provides sponsors with a high‑level overview of their activity.

**Description:**
- Summary cards showing total campaigns, active campaigns, and budget spent
- Left‑side navigation for dashboard, campaigns, influencer discovery, and logout
- Table displaying recent campaigns and their status

This screen focuses on visibility and campaign tracking for sponsors.

---

### 3. Influencer Campaign Browse Screen

**Purpose:**
- Allows influencers to discover available campaigns.

**Description:**
- Filter section for industry and budget range
- List of available campaigns displayed as cards
- Each campaign card shows sponsor name, requirements, and a call‑to‑action

This screen supports informed decision‑making by influencers before engaging in negotiations.

---

### 4. Create Campaign Screen (Sponsor)

**Purpose:**
- Enables sponsors to create new advertising campaigns.

**Description:**
- Form inputs for campaign name, description, budget, and industry
- Public/Private campaign visibility toggle
- Primary action button to create the campaign

This screen captures all essential campaign details in a structured manner.

---

### 5. Ad Request and Negotiation Screen

**Purpose:**
- Serves as the core negotiation interface between sponsor and influencer.

**Description:**
- Campaign information section with sponsor and requirement details
- Negotiation inputs for proposed payment and notes
- Status indicator (Pending / Accepted / Rejected)
- Accept and Reject action buttons

This screen formalizes the negotiation workflow and enforces mutual agreement before assignment.

---

### 6. Admin Moderation Panel

**Purpose:**
- Allows administrators to govern and moderate the platform.

**Description:**
- Tabbed navigation for Sponsors, Influencers, and Campaigns
- Search functionality
- Tabular view of entities with flag and unflag actions
- Pagination controls

This screen supports platform oversight, trust, and policy enforcement.

---

## Design Notes

- All wireframes are **low‑fidelity** and grayscale only
- Focus is on layout, hierarchy, and user flow
- No branding, colors, or animations are included
- Consistent navigation and spacing are maintained across screens

---

These wireframes collectively demonstrate the end‑to‑end functionality of IESCP and align directly with the project’s MoSCoW prioritization and implemented system behavior.
