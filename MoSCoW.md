# MoSCoW Prioritization – IESCP

This document presents the **MoSCoW prioritization** for the project **IESCP (Influencer–Sponsor Engagement & Campaign Platform)**. The prioritization is aligned strictly with the implemented scope of the system and the constraints of an academic software engineering project.

---

## Must Have
These features are essential for the system to function and directly address the core problem statement.

- User authentication with role-based access (Admin, Sponsor, Influencer)
- Account creation for sponsors and influencers
- Sponsor campaign creation, update, and deletion
- Influencer discovery of campaigns
- Structured ad request creation by sponsors
- Negotiation workflow between sponsor and influencer
- Mutual acceptance before campaign assignment
- Backend-enforced authorization and role validation
- Admin ability to view, flag, and unflag users and campaigns
- Industry categorization for sponsors and influencers

---

## Should Have
These features improve usability, visibility, and governance but are not mandatory for minimum viable operation.

- Role-specific dashboards (Admin, Sponsor, Influencer)
- Campaign status tracking (active, assigned, completed)
- Budget and spending summaries for sponsors
- Earnings and performance summaries for influencers
- Admin overview dashboard with aggregated metrics
- Export of campaign data for sponsors
- Filtering influencers by industry and reach

---

## Could Have
These features provide additional value but were deprioritized due to time and scope limitations.

- In-platform notifications for ad requests and campaign updates
- Influencer rating or feedback system
- Advanced campaign analytics
- Direct messaging between sponsors and influencers
- Saved influencer or campaign lists

---

## Won’t Have (Current Version)
These features are intentionally excluded to maintain focus and feasibility.

- Integrated payment gateway or real-time payments
- Automated influencer matching using machine learning
- Mobile application (Android/iOS)
- Multi-language support
- Integration with external social media APIs

---

## Rationale
The prioritization focuses on delivering a **stable, governed, and transparent collaboration workflow** between sponsors and influencers. Optional and advanced features were excluded to ensure clarity, maintainability, and alignment with academic project constraints.
